Bloc Jams

Bloc Jams was created based on user stories and their acceptance criteria. The finished app is a fully functional web-based music player.

Design Roles

  • UX Design
  • Front-End Web Development

Design Deliverables

  • User Stories & Acceptance Criteria (Review)
  • Finished Website

Tools Used

  • HTML & CSS (Review)
  • jQuery
  • Git & Github
Problem

Addressing the User's Needs

For this particular project, the client already had an established brand and visual design. However, they were looking to accommodate users and their desire to listen to music through functionality.

Solution

A Lightweight Web-based Product

To accommodate users, it was important to build a lightweight product. jQuery was used to add functionality to the initial visual design of the product, as it interacts easily with the DOM.

Process

User Stories

In the creation of Bloc Jams, user stories were provided from the client along with acceptance criteria which must be passed in order to address the needs of users.

As a listener, I want to see the basic information about the album so I can decide whether I'm interested in it.

Acceptance Criteria:
  • I immediately see the album cover image and title, the artist's name, and some general release info

As a listener, I want to see the album's songs and play/pause songs by clicking on them so that I can change tracks.

Acceptance Criteria:
  • I see a list of the album's songs
  • When I click a song, it plays
  • When I hover a song, it displays a "play button" in place of the song number
  • The currently playing song displays a "pause button" in place of the song number
  • A paused song displays a "play" button in place of the song number

As a listener, I want have the set standard music controls so that I have complete control over my listening experience.

Acceptance Criteria:
  • Regardless of scroll position, I see a "player bar" with control for Play/Pause, Previous Track, Next Track, Seek, and Volume
  • The Play/Pause button shows a "play" button if no song is playing, and a "pause" button if a song is playing
  • The Previous Track and Next Track buttons only respond when a song is playing
  • The Previous Track button plays the previous song on the album, but doesn't respond if there is no previous song
  • The Next Track button plays the next song on the album, but doesn't respond if there is no next song
  • I can click and drag on the time control slider to change the current song position
  • I see a slider that shows the player's current volume
  • I can click and drag the volume slider to adjust the volume

Coding the Website

Since our client already had established branding and visual design, I began by analyzing the HTML and CSS provided along with the current website’s assets. After reviewing the design of the website, I began to add functionality based on the user stories by using jQuery.


I began by tackling the first user story by creating a script to allow the user to view the album info. After, I looked to tackle the song list functionality. Last but not least, I looked to incorporate functionality to the bottom music player bar, which gave the user total control over their listening experience.

Conclusion

Key Takeaways:
  • Reviewing existing code from clients is important to see what is already implemented and what your expectations are for contribution
  • Analyzing user stories and delivering on expectations is important by passing specific acceptance criteria, especially when coding a solution
  • Use of Git/Github is key in the implementation of each user story in the process for version control and rollback capability

Bloc Jams is a project that helped me gain some valuable experience in programming using jQuery. As jQuery is a widely-accepted and popular JavaScript library, learning to utilize it will help make front-end development more efficient on other projects in the future.


This project was unique in having established branding, HTML, and CSS structure. Clients may have a well-established brand already in place, or have a website already structured where they are looking either to improve upon their current iteration, or add greater functionality via JavaScript.


In the end, my work once again brought us back to the importance of addressing the user’s experience, and making sure the product can function as intended. Through the use of jQuery, we were able to efficiently and effectively tackle their request.


In considering the future of Bloc Jams, I would focus on creating a mobile version of the web-based app. When accessing Bloc Jams on a mobile device, certain elements can be hard to access or control, especially the volume slider. With some small tweaks, this web-based app can find its way to mobile devices with ease.
